## What does the checkboxes mean?

The checklist tracks **material availability**, not attendance.

| status | meaning                         | examples                                                                                                  |
| ------ | :------------------------------ | :-------------------------------------------------------------------------------------------------------- |
| `[x]`  | Material available or verified. | Slides, code examples, official links, personal notes, demo resources and useful snippets.                |
| `[ ]`  | Material pending.               | Missing slides, unverified material, no chat link, no shared resource, needs to be requested or recreated |

A checked item means: "I have enough material to review this session later".

An unchecked item means: "I still need to find, request, validate, or recreate the material".

## Material notes convention

Notes under a session explain what kind of material exists or what is missing. Preferred wording:

```md
> Material available: slide deck.
> Material available: full code example.
> Material available: slide deck and code example.
> Material available: online slides: [Title](URL).
> Material note: live code walkthrough; no separate slide deck identified.
> Material note: no slide deck was shared. The session appears to have been delivered through full-screen sharing, so the best reference may be the meeting recording, demo content, or personal notes.
```
