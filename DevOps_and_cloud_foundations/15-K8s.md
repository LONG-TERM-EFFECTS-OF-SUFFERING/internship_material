## 🔹 Step 1: Create a Pod Running a Simple Web Server

Create a file named `pod.yaml`:

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: hello-pod
  labels:
    app: hello-app
spec:
  volumes:
    - name: html-volume
      emptyDir: {}

  initContainers:
    - name: init-html
      image: busybox
      command:
        - sh
        - -c
        - |
          echo '<html>
          <head><title>Hello from Kubernetes</title></head>
          <body>
            <h1 style="color:blue;text-align:center;">Hello, Kubernetes! 🚀</h1>
          </body>
          </html>' > /workdir/index.html
      volumeMounts:
        - name: html-volume
          mountPath: /workdir

  containers:
    - name: hello-container
      image: nginx
      ports:
        - containerPort: 80
      volumeMounts:
        - name: html-volume
          mountPath: /usr/share/nginx/html
```

Apply the Pod:

```sh
kubectl apply -f pod.yaml
```

Verify the Pod:

```sh
kubectl get pods
```

## 🔹 Step 2: Create a Deployment with a ReplicaSet

Create a file named `deployment.yaml`:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: hello-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: hello-app
  template:
    metadata:
      labels:
        app: hello-app
    spec:
      volumes:
        - name: html-volume
          emptyDir: {}

      initContainers:
        - name: init-html
          image: busybox
          command:
            - sh
            - -c
            - |
              echo '<html>
              <head><title>Hello from Deployment</title></head>
              <body>
                <h1 style="color:green;text-align:center;">Hello from Deployment!</h1>
              </body>
              </html>' > /workdir/index.html
          volumeMounts:
            - name: html-volume
              mountPath: /workdir

      containers:
        - name: hello-container
          image: nginx
          ports:
            - containerPort: 80
          volumeMounts:
            - name: html-volume
              mountPath: /usr/share/nginx/html
```

Apply the Deployment:

```sh
kubectl apply -f deployment.yaml
```

Verify:

```sh
kubectl get deployments
kubectl get replicasets
kubectl get pods
```

## 🔹 Step 3: Expose the Application with a Service

Create a file named `service.yaml`:

```yaml
apiVersion: v1
kind: Service
metadata:
  name: hello-service
spec:
  selector:
    app: hello-app
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
  type: NodePort
```

Apply the Service:

```sh
kubectl apply -f service.yaml
```

Check the NodePort:

```sh
kubectl get svc hello-service
```

Access locally:

```sh
kubectl port-forward svc/hello-service 8080:80
```

Open in browser: http://localhost:8080

## 🔹 Step 6: Create a Namespace and Move Resources

```sh
kubectl create namespace hello-namespace
kubectl get namespaces
```

---

Add `namespace: hello-namespace` under `metadata` in your YAML files.

Re-apply:

```sh
kubectl apply -f pod.yaml -n hello-namespace
kubectl apply -f deployment.yaml -n hello-namespace
kubectl apply -f service.yaml -n hello-namespace
```

---

## 🔹 Step 7: Change Default Working Namespace

Set default:

```sh
kubectl config set-context --current --namespace=hello-namespace
```

Verify:

```sh
kubectl config view --minify | grep namespace
```

Reset:

```sh
kubectl config set-context --current --namespace=default
```

## 🔍 Step 8: Intentional YAML Error Example

Let’s break a manifest to test our debugging and troubleshooting skills (and our assistant!).

Create a file named `broken-pod.yaml` with the following contents:

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: broken-pod
  labels:
    app: broken-app
spec
  containers:
    - name: broken-container
      image: nginx
      ports:
        - containerPort: 80
```

> 🧠 Can you spot the issue? Try running:

```sh
kubectl apply -f broken-pod.yaml
```

> You’ll get an error! Use the custom ChatGPT to identify what’s wrong with the YAML and how to fix it.

Once fixed, re-apply the manifest and verify the pod is running:

```sh
kubectl get pods
```
